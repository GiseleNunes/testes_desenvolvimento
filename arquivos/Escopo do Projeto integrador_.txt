Escopo do "Projeto integrador'



## 🧠 Nome do Projeto (título provocador)

**“I.A. QUE PARLA – O Robô Fofoqueiro da Acessibilidade”**  
*Ou: Como ensinar um computador a ser intérprete de surdos, mas com meme, sensor de barulho e personalidade explosiva*


## 🎯 Objetivo central (alinhado à questão motriz)

> Utilizar IA generativa + visão computacional + sensores de áudio para criar um **assistente robótico excêntrico**, que traduz sons ambientes em **alertas visuais, textuais e emocionais** (com personalidade), promovendo acessibilidade para pessoas com deficiência auditiva e mobilidade reduzida.



## 🔧 Componentes técnicos (programação + física + robótica)

### 1. **Linguagem e ferramentas**
- Python + Google Colab (para rodar em Chromebooks da rede pública)
- Bibliotecas: `SpeechRecognition`, `gTTS`, `OpenCV`, `Pandas`, `Flask` (básico), `pydub`
- Simulação de hardware: Arduino Uno + sensor de som KY-038 + LEDs RGB + display LCD 16x2

### 2. **O que o projeto faz (de forma irreverente)**
- O robô “escuta” a sala de aula.
- Se o barulho ultrapassar 75 dB → ele acende um **LED vermelho piscando** e escreve no display:
  > “TÁ ALTO, HEIN? QUER SILÊNCIO OU VOU CHAMAR A DIRETORA ROBÔ”
- Se alguém falar uma frase curta perto do microfone, ele:
  - reconhece a fala (Speech-to-Text)
  - usa um modelo de IA (ex: GPT-2 pequeno ou Gemini API free) para **reescrever a frase com meme, regionalismo ou ironia**
  - exibe no LCD a versão traduzida para **Libras textual** (ex: “BOM DIA” → “🤟 BOM DIA 🤟”)
- Se a pessoa **balançar a mão na frente da câmera** (visão computacional simples), ele:
  - lê um QR code com instruções acessíveis
  - abre um link com **mapa de acessibilidade da escola** (produzido pelos alunos)

1. **Ondulatória cômica** – medir decibéis usando um app + gritaria organizada pela turma.
2. **Ética da fofoca de IA** – será que o robô pode repetir o que ouviu? Debate sobre privacidade em Libras.
3. **UX meme-driven** – como criar interfaces para PCDs que não sejam chatas? Testar com colegas.
4. **Python para zueira** – ensinar `if` e `else` com frases do tipo:  
   `if decibeis > 80: print("CALA A BOCA AÍ, BRIGADO")`

PROMPT #2 – Sensor de som irreverente com decisões emoji

Crie um código Python para o Google Colab que simule um sensor de som (KY-038) usando o microfone do computador. O código deve:

1. Gravar 3 segundos de áudio do microfone
2. Calcular a intensidade sonora média (simular decibéis de 0 a 120)
3. Exibir no display (print formatado):
   - Nível atual: "🔇 🤫 🗣️ 📢 🔈" (emoji conforme volume)
   - Decibéis simulados (ex: 67 dB)
   - Mensagem de alerta MEME conforme tabela:
     * < 40 dB: "🤫 silêncio absoluto... até assusta"
     * 40-60 dB: "🗣️ papinho normal, nada de mais"
     * 60-80 dB: "📢 TÁ FICANDO BOM! logo logo viro fofoqueiro"
     * 80-100 dB: "🔊 CALA A BOCA! vou piscar o LED vermelho"
     * > 100 dB: "💀 INFERNO ACÚSTICO! vou chamar a diretora robô"
4. Retornar True se ultrapassou 75 dB (alerta), False caso contrário
5. Criar uma função chamada `medidor_fofoqueiro()` que faz tudo e retorna (dB, alerta, mensagem_meme)
6. Incluir um loop que repete a cada 5 segundos, com opção de apertar 'q' para sair
7. Salvar um log das medições em um arquivo CSV (timestamp, dB, mensagem)
8. Ter comentários em português com humor e referências paranaenses (ex: "# bora medir o barulho da galera")

Ao final, mostrar um gráfico simples com os últimos 10 registros usando matplotlib.
